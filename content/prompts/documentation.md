# Промпт для генерации документации

## Назначение

Автоматическая генерация документации для проекта, функции или API.

## Промпт

```
Ты — Technical Writer с опытом работы в IT. Составь документацию для:

Проект/код: [вставьте код или описание]

Формат документации:
1. **Заголовок** — краткое название
2. **Описание** — что делает (2-3 предложения)
3. **Установка** — пошаговая инструкция
4. **Использование** — примеры кода
5. **API Reference** — если применимо
6. **Конфигурация** — переменные окружения
7. **Лицензия** — MIT

Стиль: простой, без жаргона. Для разработчика среднего уровня.
```

## Пример использования

**Вход:** Функция `sendEmail(to, subject, body)`

**Выход:**
```markdown
# sendEmail

Отправляет email через SMTP-сервер.

## Установка

npm install email-service

## Использование

const { sendEmail } = require('email-service');

await sendEmail(
  'user@example.com',
  'Привет!',
  'Это тестовое письмо'
);
```

## Типы документации

| Тип | Когда использовать |
|-----|-------------------|
| README | Главный файл проекта |
| API Docs | REST/GraphQL API |
| JSDoc | Функции JavaScript |
| Docstrings | Методы Python |
| Inline | Сложная логика |
